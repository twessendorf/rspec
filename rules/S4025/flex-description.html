Having a field in a child class with a name that differs from a parent class' field only by capitalization is sure to cause confusion. Such child class fields should be renamed.

Noncompliant Code Example

public class Fruit {
  String plantingSeason;
  //...
}

public class Raspberry {
  String plantingseason;  // Noncompliant
  // ...
}

Compliant Solution

public class Fruit {
  String plantingSeason;
  //...
}

public class Raspberry {
  String whenToPlant;
  // ...
}

Or

public class Fruit {
  String plantingSeason;
  //...
}

public class Raspberry {  // field removed; parent field will be used instead
  // ...
}
